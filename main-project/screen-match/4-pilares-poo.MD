

# Em resumo sobre os 4 pilares da POO:

1. Abstração: Focar nos aspectos essenciais de um objeto, ignorando detalhes irrelevantes. Exemplo: Uma classe "Carro" pode abstrair características como "cor", "modelo" e "velocidade", sem se preocupar com detalhes internos do motor.

2. Encapsulamento: Proteger os dados internos de um objeto, permitindo acesso apenas através de métodos públicos. Exemplo: Uma classe "ContaBancaria" pode ter um saldo privado, acessível apenas por métodos como "depositar" e "sacar".

3. Herança: Permitir que uma classe herde propriedades e métodos de outra classe, promovendo reutilização de código. Exemplo: Uma classe "Cachorro" pode herdar de uma classe "Animal", adquirindo características comuns como "nome" e "idade".

4. Polimorfismo: Permitir que objetos de diferentes classes sejam tratados como objetos de uma classe comum, possibilitando o uso de métodos de forma intercambiável. Exemplo: Um método "fazerSom" pode ser implementado de maneira diferente em classes "Cachorro" e "Gato", mas ambos podem ser tratados como "Animal" ao chamar esse método.
